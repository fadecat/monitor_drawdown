# ETF轮动策略 V2 第一阶段决策文档

这份文档只记录当前已经讨论确认的 V2 第一阶段规则，目标是防止后续边聊边漂移。

更新时间：`2026-06-05`

---

## 1. 设计目标

V2 第一阶段不是直接完整照搬文章二，而是在当前 V1 的基础上，先落地一个更成熟、但仍然克制的版本：

- 保持统一规则
- 尽量少参数
- 不对特定品种做适配
- 先完成最小闭环
- 暂时不引入容易放大复杂度的风控模块

一句话概括：

**中期趋势质量排序 + 短期确认 + Top1 轮动 + 无合格标的切防守**

---

## 2. 与 V1 的核心区别

V1 的本质是：

- 风险资产和 `511880` 混在一起排名
- 用 `20` 日涨幅做唯一排名指标
- 用 `20` 日涨幅 `> 0` 做唯一过滤条件
- 多个标的合格时，直接持有 Top1

V2 第一阶段的变化主要有三条：

1. 防守资产从普通候选池中分离出来
2. 排名从简单区间涨幅升级为趋势质量评分
3. 增加一个短期确认过滤

---

## 3. 资产池结构

### 3.1 风险资产池

风险资产池参与：

- 技术指标计算
- 候选过滤
- 排名
- Top1 选择

### 3.2 防守资产池

防守资产池不参与风险资产之间的竞争。

它只在下面这种情况下接管仓位：

- 风险资产池里没有任何合格标的

当前默认防守资产：

- `511880` 银华日利ETF

### 3.3 明确规则

- 风险资产合格：进入排名池
- 风险资产全不合格：切换到防守资产
- 防守资产不与风险资产比较谁更强

这一步的本质不是调参，而是把资产角色分清。

---

## 4. 排名函数定义

V2 第一阶段采用文章二的核心思路。

### 4.1 主窗口

- `lookback_days = 25`

### 4.2 输入序列

- 使用最近 `25` 个交易日的 `close`
- 先按当前研究口径实现
- 暂不要求 `OHLC`

### 4.3 计算流程

设价格序列为：

- `price_series = [p1, p2, ..., p25]`

先做对数变换：

- `y = log(price_series)`

横轴：

- `x = [0, 1, 2, ..., 24]`

权重：

- `weights = linspace(1, 2, 25)`

然后做加权一元线性回归，得到：

- `slope`
- `intercept`

再计算：

- `annualized_return = exp(slope * 250) - 1`
- `ss_res = sum(weights * (y - y_hat)^2)`
- `ss_tot = sum(weights * (y - mean(y))^2)`
- `r_squared = 1 - ss_res / ss_tot`

最终分数定义为：

- `score_25 = annualized_return * r_squared`

这里明确约定：

- `slope` 用加权回归得到
- `r_squared` 也使用同一组 `weights` 计算
- 不使用普通未加权 `R²`

### 4.4 这个分数的含义

- `annualized_return`：最近这段趋势涨得快不快
- `r_squared`：最近这段趋势走得稳不稳
- `score_25`：只奖励“涨得快且涨得稳”的标的

### 4.5 为什么保留加权回归

权重从 `1` 到 `2` 线性递增，表示：

- 最近数据更重要
- 但不会完全忽略更早的数据

这不是为了贴合某个品种，而是为了让排序更接近“当前仍在发力的强者”。

## 5. 候选过滤条件

V2 第一阶段只采用两条过滤条件。

### 5.1 条件一：中期趋势必须为正

- `score_25 > 0`

含义：

- 最近 `25` 日整体趋势必须向上

边界处理约定：

- 如果 `R² < 0`，按 `0` 处理
- 此时 `score_25 = 0`
- 因为过滤条件是 `score_25 > 0`，所以该标的直接淘汰

### 5.2 条件二：短期不能已经转弱

- `return_10d > 0`

其中：

- `return_10d = close[t] / close[t-10] - 1`

含义：

- 最近 `10` 日必须仍然处于正收益状态

### 5.3 过滤顺序

建议实现顺序：

1. 计算 `score_25`
2. 过滤 `score_25 > 0`
3. 再过滤 `return_10d > 0`
4. 剩余标的进入最终排名

### 5.4 第一阶段暂不加入的过滤条件

先不加入：

- `RSI`
- `min_score_threshold`
- `max_score_threshold`
- 其他更复杂的二次过滤

理由是：

- 先把排序层和防守层跑顺
- 便于观察哪一层真正带来改善

---

## 6. 选仓规则

### 6.1 持仓数

- `holdings_num = 1`

### 6.2 多个标的都合格时

- 无条件持有排名第 `1` 的标的
- 不做“减少换仓”的缓冲规则
- 不做“只有领先多少才换仓”的阈值

### 6.3 当前共识

在当前讨论范围内：

- ETF 交易费用较低
- 这一阶段不考虑手续费和滑点

因此：

- 如果 Top1 改变，就直接切换

---

## 7. 防守规则

如果风险资产池里：

- 没有任何标的同时满足 `score_25 > 0` 和 `return_10d > 0`

则：

- 不空仓
- 直接切换到防守资产 `511880`

### 7.1 当前第一阶段只保留这一层防守

V2 第一阶段不加入：

- 固定止损
- ATR 跟踪止损
- RSI 过热过滤

也就是说：

- 当前唯一的“退出风险资产”机制，是风险池重新筛选后发现没有合格标的

---

## 8. 第一阶段明确不做的内容

为了避免范围失控，下面这些内容明确放到后续阶段：

- `RSI(6) <= 95`
- 固定止损 `0.95`
- `2*ATR` 跟踪止损
- `loss = 0.97` 这类额外风控参数
- 持仓滞后缓冲，例如 `top1 threshold`
- 手续费
- 滑点
- 更真实的开盘成交假设
- 为特定品种做参数适配

## 9. 执行时序与回测记账

这一部分在 V2 第一阶段正式冻结，不再保留候选方案。

### 9.1 基本口径

- `signal_date` 表示 `T` 日收盘
- `signal_date` 当天只负责算信号，不负责成交
- `position_date` 表示全局日期序列中的下一个交易日，也就是 `T+1`
- 当前阶段继续采用研究口径，不做严格开盘成交模型

一句话概括：

- `T` 日收盘算信号，`T+1` 日按收盘价记执行收益

### 9.2 全局日期序列

V2 第一阶段继续沿用简化但统一的研究框架：

- 回测日期轴使用全部已加载品种交易日期的并集
- `position_date` 取这个全局日期轴中 `signal_date` 的下一个日期
- 起始信号日取“所有非空序列都已经满足回看窗口要求”的最早日期

这样做的目的不是追求最真实，而是：

- 保持所有资产在同一研究时钟下比较
- 先把 V2 排名与防守逻辑单独验证清楚

### 9.3 单日收益如何记

若 `signal_date = T`，`position_date = T+1`，且选中标的是 `X`，则：

- 用 `X` 在 `T` 日的收盘价计算信号
- 用 `X` 从 `T` 日收盘到 `T+1` 日收盘的涨跌幅作为该次持仓的单日收益

公式写法：

- `daily_return = close[T+1] / close[T] - 1`

对应到输出文件：

- `daily_positions.csv` 的 `date` 字段表示 `position_date`
- `daily_positions.csv` 必须额外保留 `signal_date`

### 9.4 风险资产与防守资产在时序里的角色

- 风险资产只参与 `signal_date` 当天的过滤和排名
- 若风险资产存在合格 Top1，则 `T+1` 持有该风险资产
- 若风险资产全部不合格，则 `T+1` 切到防守资产

### 9.5 明确限制

这里必须明确：

- 这是研究记账口径，不是严格可成交口径
- 还没有引入开盘价、盘口、滑点、申赎冲击等现实执行细节
- 若后续进入更接近实盘的阶段，再单独升级执行模型

## 10. 输出契约

V2 第一阶段不只要给一个最终结论，还要把中间过程暴露出来，便于复核与排错。

### 10.1 Runner 输出

运行单次最新信号时，至少输出：

- `candidate_metrics.json`
- `ranked_candidates.json`
- `portfolio_decision.json`
- `summary.md`

其中：

- `candidate_metrics.json` 保存全部风险资产的计算结果，包括未入选标的
- `ranked_candidates.json` 只保存通过过滤后的风险资产，并按分数降序排列
- `portfolio_decision.json` 保存最终持仓决策，包含是否触发防守
- `summary.md` 用人能直接看的方式概括结果

### 10.2 Backtest 输出

运行回测时，至少输出：

- `daily_candidate_metrics.csv`
- `daily_rankings.csv`
- `daily_positions.csv`
- `trades.csv`
- `holding_periods.csv`
- `yearly_returns.csv`
- `symbol_contributions.csv`
- `backtest_summary.md`

### 10.3 关键字段不能省

无论是单次运行还是回测明细，下面这些字段都必须能看到：

- `score_25`
- `annualized_return_25`
- `r_squared_25`
- `return_10d`
- `selection_reason`
- `reason`

### 10.4 字段的用途

- `score_25`：看最终排名到底由谁赢
- `annualized_return_25`：看“快不快”
- `r_squared_25`：看“稳不稳”
- `return_10d`：看短期确认是否还成立
- `selection_reason`：区分是风险资产入选，还是切到了防守资产
- `reason`：区分这次交易属于首次建仓、风险资产轮换、切入防守，还是从防守重新进攻

### 10.5 `trades.csv.reason` 枚举

V2 第一阶段将 `trades.csv` 的 `reason` 固定为下面五类：

- `initial_entry_risk`
- `initial_entry_defensive`
- `risk_to_risk_rotation`
- `risk_to_defensive_fallback`
- `defensive_to_risk_reentry`

## 11. V2 第一阶段最小闭环

如果只看这一阶段最终落地内容，策略可以写成：

1. 每个交易日收盘后，拿风险资产池最近 `25` 日收盘价
2. 计算每个标的的 `score_25 = annualized_return * r_squared`
3. 再计算 `return_10d`
4. 只保留满足以下条件的标的：
   - `score_25 > 0`
   - `return_10d > 0`
5. 对合格标的按 `score_25` 从高到低排序
6. 如果至少有一个合格标的：
   - 持有排名第 `1` 的风险资产
7. 如果一个都没有：
   - 切换到 `511880`
8. 下一交易日执行调仓
